# The Deep Past Challenge - 競賽概述

## 概述

The Deep Past Challenge 提出了一個大膽的問題：**AI 能否解碼 4000 年前的商業記錄？**

在本競賽中，你將協助解碼古亞述商人的日常商業記錄。使用來自 8,000 份楔形文字文本的資料，你的目標是建立一個古亞述語（Old Assyrian）的翻譯系統。全世界博物館的抽屜裡還有數千份類似的文本尚未被閱讀。

**你的工作可以幫助將他們的聲音帶回人類的歷史故事中。**

---

## 競賽說明

四千年前，亞述商人留下了世界上最豐富的日常生活和商業活動檔案之一。數以萬計的泥板記錄著債務清算、商隊派遣，以及日常家庭事務的討論。今天，這些泥板中有一半仍然沉默，不是因為它們已損壞，而是因為很少人能讀懂壓印在泥土上的語言。許多泥板在博物館抽屜中未被翻譯已超過一個世紀。

The Deep Past Challenge 將這個古老的謎團轉變為現代機器學習問題，邀請參賽者協助解鎖古代世界最大的未翻譯檔案。我們邀請你為古亞述楔形文字泥板建立翻譯模型：這些青銅時代的文本在博物館藏品中未被閱讀已超過一個世紀。古亞述語——這些泥板上使用的方言——是阿卡德語（Akkadian）的早期形式，也是最古老有記錄的閃米特語。

現存有近兩萬三千份泥板記錄著連接美索不達米亞與安納托利亞的古亞述貿易網絡。其中只有一半被翻譯，而全世界專門研究其餘部分的學者不到十幾位。

這些不是希臘和羅馬那些經過潤飾的經典作品，由抄寫員精選和複製，決定誰的聲音能夠流傳。這些是未經過濾的，直接來自撰寫者本人的：古代商人及其家人用泥土寫成的信件、發票和合約。**它們就像青銅時代的 Instagram 限時動態：平凡、即時、且令人驚嘆地真實。**

### 挑戰內容

你的任務是建立神經機器翻譯模型，將轉寫的阿卡德語（transliterated Akkadian）轉換為英語。挑戰在於：阿卡德語是一種低資源、形態複雜的語言，其中一個單詞可以編碼英語中需要多個單詞才能表達的內容。為現代、資料豐富語言建立的標準架構在這裡會失敗。

破解這個問題，你將為 10,000+ 份未翻譯的泥板賦予聲音。而且你不僅僅是復興過去：你將協助開創一個翻譯藍圖，用於數千種 AI 時代尚未觸及的瀕危和被忽視語言——無論是古代還是現代的。

**了解更多：** 訪問[此網站](https://www.deeppastinitiative.org/)以了解更多關於組織團隊、Deep Past Initiative 以及更多背景資料。

## 評估標準

提交作品將通過 **BLEU 和 chrF++ 分數的幾何平均值**進行評估，每個分數的充分統計量會在整個語料庫中彙總（即每個分數是微平均值）。

你可以參考 [SacreBLEU 函式庫](https://github.com/mjpost/sacrebleu)了解實作細節。可在 Kaggle 上找到實作該指標的筆記本：[BLEU 和 chrF++ 的幾何平均值](https://www.kaggle.com/code/ryanholbrook/geometric-mean-of-bleu-and-chrf)。


## 資料集說明

處理阿卡德語 / 古亞述文本時，最大的挑戰是處理格式問題。正如俗話所說，**「垃圾進，垃圾出」**，不幸的是，轉寫文本的格式在機器學習工作流程的每個步驟（從分詞到轉換和嵌入過程）都帶來了挑戰。

為了緩解這些問題，我們提供以下資訊和建議，以處理轉寫和翻譯文本中的不同格式挑戰。

### 轉寫文本（Texts in Transliteration）

**主要格式挑戰：** 除了標準的轉寫格式（帶有連字符的音節），額外的書記員註解使文本充斥著上標、下標和標點符號，這些符號只對亞述學專家有意義（[完整轉寫轉換指南](https://cdli.mpiwg-berlin.de/articles/cdli/transliteration)）。

#### 大小寫（Capitalization）

大小寫以兩種不同方式編碼意義：
- **首字母大寫** → 表示該詞是人名或地名（專有名詞）
- **全部大寫** → 表示它是蘇美語標誌符號（logogram），是為了書寫簡便而代替阿卡德語音節拼寫

#### 限定符（Determinatives）

限定符在阿卡德語中用作名詞和專有名詞的分類符號。這些符號通常以上標格式列印在它們分類的名詞旁邊。為了避免將限定符號誤讀為單詞的一部分，我們遵循標準轉寫指南並在其周圍保留了大括號。雖然這可能對機器學習構成挑戰，但我們注意到這是轉寫中大括號的唯一用途（例如 `a-lim{ki}`、`A-mur-{d}UTU`）。

#### 破損文本（Broken Text）

由於這些是古代文本，它們包含許多破損和缺失。為了標準化這些破損，我們建議只使用兩個標記：
- `<gap>` - 用於單個符號的小破損
- `<big_gap>` - 用於超過一個符號到大型破損

為了本挑戰的目的，我們在下面提供了如何最佳處理這些格式問題的建議。

### 翻譯文本（Texts in Translation）

目前還沒有完整或廣泛的古代楔形文字文件翻譯資料庫，尤其是古亞述文本。因此，我們收集了包含古亞述文本翻譯和評論的書籍和文章，並使用 OCR 和 LLM 進行數位化和修正。即使經過所有這些工作，這些翻譯仍存在許多格式問題，這使其成為成功開發機器翻譯的核心組成部分。

翻譯通常保留相同的專有名詞大寫，而這些專有名詞通常是大多數機器學習任務表現不佳的地方。為了解決這些問題，我們在資料集中包含了一個詞典，其中包含所有專家為印刷出版物規範化的專有名詞。

### 現代書記員註記（Modern Scribal Notations）

重要的是要注意，轉寫和翻譯文本附帶有現代書記員註記。其中第一種包括行號。這些通常編號為 1、5、10、15 等。但是，如果有任何破損的行，行號後面會立即跟隨一個撇號（`'`），如果有第二組破損的行，行號將有兩個尾隨撇號（`''`）。這些不是引號，而是編輯有時在出版物中使用的書記員慣例。

#### 額外的書記員註記

| 符號 | 意義 |
|------|------|
| `!` | 確定讀取困難符號 |
| `?` | 不確定讀取困難符號 |
| `/` | 屬於某行的符號位於該行下方 |
| `:` | 古亞述單詞分隔符號 |
| `( )` | 破損和擦除的註釋 |
| `< >` | 進行更正時的書記員插入 |
| `<< >>` | 錯誤或有誤符號的標記 |
| `˹ ˺` | 部分破損符號的半括號 |
| `[ ]` | 明顯破損符號和行的方括號 |
| `{ }` | 限定符的大括號 |

### 格式建議

#### 移除（現代書記員註記）

- `!` (確定讀取)
- `?` (可疑讀取)
- `/` (行分隔符)
- `:` 或 `.` (單詞分隔符)
- `< >` (書記員插入，但保留轉寫/翻譯中的文本)
- `˹ ˺` (部分破損符號)
- `[ ]` (從文件層級轉寫中移除。例如 `[KÙ.BABBAR]` → `KÙ.BABBAR`)

#### 替換（破損、缺失、上標、下標）

- `[x]` → `<gap>`
- `…` → `<big_gap>`
- `[… …]` → `<big_gap>`
- `ki` → `{ki}` (見下方完整列表)
- `il5` → `il5` (對任何下標數字都相同)
